I’m the Dev Beaver and am happy to welcome you on Hoodie’s Code Team.

Our main objective is to make Hoodie’s code as easy to contribute to as possible. If there is something that doesn’t make sense, it’s on us, please let us know and we will try to clarify it for you and document it for everyone else for the future.

Our Code Team lead is Gregor. He keeps the overview of everything code and helps everyone interested to contribute to or help maintain Hoodie’s codebase. Say hi on twitter or on our Hoodie Chat.

What is Hoodie?

Hoodie, at its core, is a server that comes with its own database and a client API. It basically allows you to build web applications with just HTML, CSS and JavaScript, and use the Hoodie Client AP for user authentication and data persistance.

If you want to understand how things work internally, a good place to start is the Hoodie Server README. But don’t get intimidated by all of Hoodie’s code. We worked hard to split up the code base into smaller modules, and if you want to start to contribute, you can start contributing to each of this small modules, without the need to understand all of Hoodie. Each module has their own documentation, tests and instructions on how to get going.

How to contribute?

We have a list of prepared code issues on our camp repository. If you want to jump right in, these are for you :) Some of them are reserved for first-time Hoodie Contributors. We also list the issues below.

We documented all our conventions on contributing code to Hoodie, and if you continue contributing, we’d like you to read through it. But if you are new to Hoodie, please don’t worry, start your first pull request, we are more than happy to help you out with things like code style. Don’t let it keep you from contributing in the first place, we are here to help.
